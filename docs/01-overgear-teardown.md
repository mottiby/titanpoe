# Документ 1. Разбор overgear.com (teardown)

> Цель документа: разобрать референс по фактам, чтобы ТЗ и архитектура нашего PoE2-маркетплейса строились на реальной модели, а не на догадках. Данные собраны с overgear.com, help.overgear.com и work.overgear.com (см. «Источники»).

---

## 1. Что такое overgear

Международный **маркетплейс игровых услуг**: 55+ игр (World of Warcraft и его варианты, **Path of Exile 2**, Final Fantasy XIV, Destiny 2, Diablo 4, Valorant, CS2, League of Legends и др.). 5000+ отобранных исполнителей. Репутация подтверждается через Trustpilot.

Суть: площадка-посредник между **покупателями** услуг и независимыми **исполнителями** (бустеры, поставщики валюты, команды), которая берёт на себя оплату, гарантию и разрешение споров за **комиссию**.

## 2. Бизнес-модель (ядро)

| Параметр | Как у overgear |
|---|---|
| Тип | Курируемый / platform-mediated маркетплейс (не открытая доска) |
| Кто продаёт | Независимые исполнители, прошедшие отбор/KYC/интервью |
| Роль платформы | Витрина + эскроу-гарант + модерация + саппорт + споры |
| **Монетизация** | **Комиссия 12% базовая, до 18%** в отдельных случаях + платные доп-опции/продвижение |
| Деньги | Холдятся в **эскроу**; релиз исполнителю — после подтверждения покупателя |
| Авто-релиз | Если покупатель молчит **72 ч** после загрузки доказательств — сделка авто-закрывается |

Вывод для нас: это **C2C-маркетплейс с сильной курацией и эскроу**, а не B2C-магазин и не свободная барахолка. Эту модель и берём.

## 4. Роли и акторы системы

- **Покупатель (buyer)** — заказывает услугу/товар.
- **Продавец/исполнитель (seller)** — подтип: booster / currency supplier / team. Выставляет листинги с ценой, имеет рейтинг/отзывы/скорость.
- **Платформа** — каталог, эскроу, модерация, споры, саппорт.
- **Модератор/саппорт** — верификация продавцов, разбор споров (help.overgear.com).
- *(потенциально)* **Аффилиат/реферал** — на overgear есть отдельные программы.

## 5. Путь покупателя (order flow) — 4 шага

1. **Browse** — выбирает услугу/товар; видит список продавцов с ценой, рейтингом, скоростью исполнения.
2. **Book with payment** — оплачивает; деньги уходят в **эскроу** («на удержание в безопасном месте»).
3. **The Magic Begins** — открывается **чат с продавцом** для согласования (расписание, данные аккаунта); исполнитель работает в срок.
4. **Confirm Completion** — продавец грузит доказательства; покупатель подтверждает → деньги релизятся. Нет реакции 72 ч → авто-подтверждение.

Защита: **100% Moneyback Guarantee** + «deals protection» (получишь оплаченное либо возврат) + dispute/refund policy.

## 6. Онбординг продавца (seller pipeline)

`register account` → **KYC** (паспорт-ID + телефон) → **интервью** (проверка игровых и коммуникационных навыков) → роль обновляется на **Seller** → создаёт листинги. Отдельный портал найма: work.overgear.com. Под отдельные потоки (валюта/голд) — отдельные формы-заявки.

Вывод для нас: продавец — это **не самозапись в один клик**. Нужен пайплайн: заявка → верификация личности → ручная/полу-ручная модерация → активация.

## 7. Доверие и безопасность (trust & safety)

- **ID-verified players** — отметка верификации в выдаче.
- **Рейтинги, отзывы, скорость** — на каждом продавце.
- **Trustpilot** — внешний соц-пруф.
- **Гарантия возврата + эскроу** — финансовая защита покупателя.
- **Споры/рефанды** — формализованная политика + саппорт.
- Страница «Guarantees» и «User Agreement» (юр. рамка).

## 8. Платежи

Принимаются: Mastercard, Visa, PayPal, Apple Pay, Google Pay, JCB, **Stripe**, SEPA. Наличие **Stripe** в стеке overgear — сильный аргумент в пользу **Stripe Connect** для нашей эскроу-логики и сплита выплат продавцам.

## 9. Способы доставки услуги (fulfillment) — критично для PoE2

Разные категории = **разные потоки исполнения и разные риски**:

- **Selfplay / party-play** — бустер играет вместе с покупателем (без передачи аккаунта). Безопаснее по ToS.
- **Piloting / account-sharing** — бустер заходит на аккаунт покупателя. Удобнее, но выше риск (бан/ToS/безопасность).
- **Currency / items (trade)** — передача в игре через трейд; почти мгновенно; риск RMT-детекта.
- **Coaching** — созвон/сессия; без доступа к аккаунту; минимальный риск.

Это прямо влияет на модель данных (у заказа есть `fulfillment_type`) и на UX (разные чек-листы, разные дисклеймеры).

overgear — широкий (55+ игр), мы — **глубокий по одной игре**:
