# Документ 4. Дорожная карта (MVP → полная версия)

**Опирается на:** [PRD](02-product-spec-prd.md) и [архитектуру](03-technical-architecture.md).
**Принцип:** **вертикальные срезы** — каждая фаза даёт работающий сквозной кусок ценности; порядок — по зависимостям, не по «важности». Детальная разбивка на задачи внутри фазы — на этапе кода (скилл `planning-and-task-breakdown`).

---

## Логика очерёдности
Сначала строим **механику маркетплейса** на самой простой категории (**Currency/trade** — почти мгновенная доставка), затем расширяем категориями, потом монетизация/рост, и в самом конце — **рискованные потоки** (piloting, продажа аккаунтов) после юридической проработки.

```
Фаза 0 ──> Фаза 1 ──> Фаза 2 ──> Фаза 3 ──> Фаза 4 ──> Фаза 5
Фундамент   MVP       Бустинг    Предметы   Рост/      Рискованное
           (trade+    (party-    +крафт     монетиза-  (piloting,
           coaching)  play)                 ция        аккаунты)
```

## Фаза 0 — Фундамент (без публичного релиза)
**Цель:** каркас, на котором можно строить.
- Репозиторий, стек, CI (lint/typecheck/test), окружения, Sentry.
- Prisma-схема + миграции + сиды (категории, лиги PoE2).
- Аутентификация (email/OAuth/2FA) + роли (RBAC).
- i18n EN/RU, базовый layout + дизайн-система (shadcn).
- Скелет интеграций: Stripe Connect + Identity (test mode), Inngest, Resend, Pusher, R2.
- ⚠️ **Блокер до денежного кода:** подтвердить допустимость Stripe для RMT-категорий (или выбрать PSP).

**Exit:** можно зарегистрироваться, стать продавцом (KYC test), создать листинг; CI зелёный.

## Фаза 1 — MVP (первый публичный релиз)
**Категории:** Currency (trade) + Coaching (session) — самые простые потоки.
**Сквозной цикл:** каталог+фильтры → выбор продавца → заказ → оплата+эскроу → чат с пруфами → подтверждение / авто-релиз 72ч → отзыв.
- Жизненный цикл заказа (машина состояний) + эскроу: релиз/возврат.
- Базовые споры + очередь модерации (админка).
- Выплаты продавцу (payout) + удержание комиссии **15%**.
- Trust: рейтинги, бейдж верификации, гарантия и политика возврата.

**Exit (Definition of Done MVP):** реальная сделка end-to-end на Stripe **live** (малой суммой) проходит; спор разрешается; выплата уходит продавцу; ошибки/метрики видны в Sentry.

## Фаза 2 — Бустинг (party-play)
**Категории:** Power Leveling, Boss Carries, Atlas/Maps farming, Challenge completion (всё **без передачи аккаунта**).
- Спец-фильтры под контент (босс, уровни, прогресс Атласа).
- Пруф-флоу и согласование расписания под многочасовые бусты.

**Exit:** заказ буста party-play проходит полный цикл с пруфами и релизом.

## Фаза 3 — Предметы и крафт
**Категории:** Items/Gear, Crafting on demand.
- Атрибуты предметов, флоу «кто даёт материалы», поэтапная приёмка крафта.

## Фаза 4 — Рост и монетизация
- Featured/boosted листинги (платное продвижение).
- Pro-подписка для продавцов (ниже комиссия / больше листингов).
- Поиск Meilisearch/Typesense + расширенные фильтры.
- Контент/SEO: гайды по лигам/боссам/экономике (органический трафик — наш дифференциатор).
- Аналитика, реферальная программа.

## Фаза 5 — Рискованные потоки (только после легал/ToS-проработки)
- **Piloting** (account-sharing): безопасная передача доступа, усиленные дисклеймеры/модерация, рекомендация смены пароля.
- **Продажа аккаунтов:** отдельная модерация и проверка.
- **Гейт:** решены вопросы ToS / PSP / юрисдикции из [PRD §9](02-product-spec-prd.md) и [арх. §13](03-technical-architecture.md).

## Контрольные точки и риски
| Риск | Где | Митигация |
|---|---|---|
| Stripe запретит RMT | до Фазы 1 | проверить заранее; запасной gaming-PSP |
| Чарджбеки / фрод | Фаза 1+ | эскроу, KYC, резервы, лимиты новичкам |
| Сложность для соло | весь проект | managed-сервисы, вертикальные срезы, узкий MVP |
| Баны игроков (ToS) | Фазы 2, 5 | party-play first, дисклеймеры |

## Первые 5 шагов (когда начнём код)
1. Инициализировать Next.js + TS + Tailwind + shadcn + Prisma + i18n.
2. Поднять managed Postgres, схему, сиды категорий/лиг.
3. Auth + роли + кабинет.
4. Stripe Connect онбординг продавца (test) + создание листинга.
5. Сквозной заказ Currency: оплата → эскроу → подтверждение → релиз (test mode).
